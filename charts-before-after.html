Соответствие типографике (Typography v1.0)

Все размеры и веса в графиках берутся из шкалы типографики — никаких новых значений вне согласованной системы.

  • Datalabels (значения над/в столбиках, на линиях, имена source у пузырей): --fs-caption = 11px / regular (через Chart.defaults.plugins.datalabels.font.size = 11). Никакого weight: 'bold' — если хочется выделить, используем цвет (--c-growth, --tx-ondark на тёмном фоне), но не толщину.
  • Подписи осей и легенды: --fs-chart-meta = 10px / regular (через Chart.defaults.font.size = 10 и plugins.legend.labels.font.size = 10).
  • Sparkline min/max: --fs-chart-meta = 10px / regular. Цвета — --c-growth для max, --c-decline для min (выделение цветом, не размером и не весом).
  • Шрифт: 'Ubuntu Mono' везде, через Chart.defaults.font.family.

1. Форматтеры — четыре функции, единые для всех

Заменяют 11+ разных вариантов. Все автоматически скрывают 0 и null. Локаль ru-RU для разделителя тысяч.

Функция Вход 0 Вход 12345 Вход 12.456 Вход null
fmt.num '' 12 345 12.456 ''
fmt.pct '' 12345.0% 12.5% ''
fmt.sec '' 12345s 12s ''
fmt.cur '' $12 345 $12 ''

2. Маркеры легенды — компактные

Сейчас Chart.js по умолчанию рисует прямоугольник ~36×12px — крупно, ест место в шапке графика. Применяем правило по типу графика: для bar — квадрат 10×10, для line — короткая линия (20×2 = «два квадрата в ширину»), для bubble/sparkline — круг 10×10.

Тип графика Маркер Размер / форма Chart.js конфиг
Bar (любой) Квадрат 10×10 usePointStyle:true · pointStyle:'rect' · boxWidth:10
Line · Combo line-часть Линия 20×2 (≈2 квадрата) usePointStyle:true · pointStyle:'line' · boxWidth:20
Bubble Круг 10×10 usePointStyle:true · pointStyle:'circle' · boxWidth:10
Sparkline — нет легенды — display:false legend.display:false
Сейчас (по умолч.) Прямоугольник ≈36×12 — дефолт Chart.js —

4. Дополнительные правила (новые)

Три темы, которые ты поднял после первого прохода: высота графиков, ширина баров, sparkline с min/max и сеткой. Стандартные паттерны из аналитических дашбордов — добавляю в спецификацию.

4.1 Высота графиков — четыре ступени

Вместо «у каждого блока своя случайная высота» — фиксированные ступени, выбираются по контексту. Реализуется CSS-переменными --ch-xs / --ch-sm / --ch-md / --ch-lg; внутри контейнера ставим min-height и max-height на одно и то же значение, чтобы график не «дышал» при разных данных.

Ступень Размер Где использовать Примеры в дашборде
--ch-xs 60px Sparkline в KPI-карточках .kpi-spark
--ch-sm 180px Графики в слайдер-карточках (компакт) Каналы slider, source slider
--ch-md 280px Стандартный график в .cell Воронка, тренды конверсии
--ch-lg 360px Детализированные с осями + большая легенда Bubble-чарты, dual-axis combo

Визуально — четыре одинаковых графика, каждый в своей ступени. Видно как меняется «вес»:

--ch-xs · 60px
--ch-sm · 180px
--ch-md · 280px
--ch-lg · 360px

4.2 Ширина баров — maxBarThickness

Когда периодов мало (например, 4 недели в месячном отчёте) — Chart.js по умолчанию растягивает бары на всю ширину, они выглядят «жирно» и не аналитично. Решение: maxBarThickness: 40 — бар не может быть шире 40px, после порога появляются зазоры справа. Это стандартная практика для дашбордов.

Сейчас (4 недели, бары растянуты)
С maxBarThickness: 40
  • Каждый dataset получает maxBarThickness: 40 — это «потолок»; если периодов много, бары всё равно становятся уже автоматически
  • Минимальная ширина регулируется через categoryPercentage: 0.8 и barPercentage: 0.9 (стандарт Chart.js — даёт небольшой зазор между группами)
  • Что делать когда баров слишком много (например, 30 дней в day-grain):
    • → Первый путь — переключатель grain (у нас уже есть): пользователь жмёт «week» и получает 4-5 баров вместо 30
    • → Второй путь (опционально) — горизонтальный скролл контейнера: overflow-x: auto + минимальная ширина графика. Применять только там, где агрегация невозможна

4.3 Sparkline — min/max и лёгкая сетка

Sparkline — это «тренд глазами», но без чисел его сложно прочитать. Добавляем две тихие подписи у крайних точек (max и min) и лёгкую вертикальную сетку по периодам — становится понятно «вот здесь была пиковая неделя».

Сейчас
Только линия. Не видно ни значений, ни границ периодов
Стало — min/max подписи + лёгкая сетка
220 и 120 — макс и мин (9px, цвет по семантике); вертикальные линии — недели
  • Min/max подписи: только в крайних точках через datalabels.display: ctx => v === max || v === min
  • Цвет подписей: max → --c-growth (зелёный), min → --c-decline (красный) — даёт быстрый сигнал «куда смотреть»
  • Размер подписей: 9px (мельче --fs-chart-meta, чтобы не доминировали над линией)
  • Сетка: x.grid.color: 'rgba(0,0,0,0.06)' — едва заметна, но позволяет «оценить когда»
  • Tooltip остаётся выключенным (enabled:false) — sparkline не для интерактива
