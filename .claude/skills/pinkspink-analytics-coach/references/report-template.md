# Report Templates

Three templates: daily, weekly, monthly. All delivered via Telegram Bot API (`parse_mode=Markdown` v1).

## Telegram formatting rules

- `*bold*` — bold (NOT `**bold**`, that's MarkdownV2 and will break parsing)
- `_italic_`
- `` `inline code` ``
- ` ``` ` code block ` ``` `
- **NO Markdown tables** (`|...|` syntax) — Telegram does not render them. Use the inline list patterns below.
- Avoid bare `[text](url)` links and `>` blockquotes in body — they can cause parse errors and the message falls back to unstyled plain text.
- Bullet points: use `—` or `·` as visual separators, not `*` or `-` (those have Markdown meaning).

## Daily template

### Quiet day (≤30 words, no sections)

Use when nothing significant moved: sessions within ±25% of trailing 7-day average, ATC events at baseline level, no unusual country spikes.

```
*[DD.MM] — спокойный день*

[N] сессий, всё в пределах нормы. [Опционально одна строка: чего ждём. Если нечего — пропустить.]
```

Nothing else. No blocks, no bullets, no "смотрим завтра".

---

### Day with event (≤200 words of prose)

```
*[DD.MM]*

*[Headline 1–2 фразы. Что произошло простым языком + привязка к нашим действиям, если есть.]*

*Корзины:*
[DD.MM] 🇯🇵 Город · Канал/source · view → ATC → checkout
[DD.MM] 🇺🇸 Город · Канал/source · view → ATC
...

[Если есть excluded/сомнительные — одна строка под списком: "⚠️ Тбилиси — скорее всего ассистент. Реальных клиентских корзин N."]

*Трафик:* 🇯🇵 Япония N · 🇺🇸 США N · 🇩🇪 Германия N · ... (5–7 стран через ·)

*Что важно:*
— [1–2 предложения с цифрами и привязкой к рекламе/changelog]
— [следующий пункт]
— [до 4 пунктов]

*Смотрим завтра:*
— [конкретный индикатор]
— [ещё один, не больше 3]
```

## Weekly template (≤700 words)

```
*Прошлая неделя · [start_date] – [end_date]*

*[Headline 1–2 фразы. Главный сдвиг + связь с нашими действиями.]*

*Цифры:*
Сессии: N (норма M, ±X%)
Корзины: N (норма M, ±X%)
Checkout: N (норма M, ±X%)
Покупки: N

*Корзины ([N] человек):*
[DD.MM] 🇰🇿 Астана · Social/ig · view → ATC → checkout
[DD.MM] 🇯🇵 Нагоя · Social/ig · view → ATC → checkout
[DD.MM] 🇯🇵 Сибуя · Direct · view → ATC
...

[1 предложение под списком: сколько из Японии, сколько органика, дошёл ли кто до покупки.]

*Ритм недели:*
```
27.04 пн:  16
28.04 вт:  17
29.04 ср:  19
30.04 чт:  87 ← реклама + IG-bio
01.05 пт:  84
02.05 сб:  76
03.05 вс: 100
```
[1 предложение про паттерн.]

*Откуда трафик:* 🇯🇵 Япония N (X%) · 🇺🇸 США N · 🇩🇪 Германия N · ... · остальные N стран

*Каналы:*
Social ±X% [← одна фраза если есть объяснение]
Paid ±X% [← Stage-2 / не крутится и т.д.]
Direct ≈ флэт
Organic ±X%
Referral ±X%

*Что важно:*
— [главный инсайт: 1–2 предложения, цифры, привязка к рекламе/changelog]
— [следующий]
— [до 4 пунктов]

*Кандидаты на расширение:*
🇺🇸 США (Сиэтл) · 26 сес · 1 ATC · единственный non-JP город с действием
🇰🇿 Казахстан · 2 сес · 1 ATC+checkout · оба покупательных события за историю
[Если нет кандидатов — одна строка: "Сегментов с устойчивым сигналом качества на этой неделе не найдено."]

*Рекомендации:*
1. *[Что сделать]* — [1–2 предложения. Мотивация. Бюджет/время. Метрика.]
2. ...
3. ...
[Максимум 3. Если нечего — "Конкретных рекомендаций нет, продолжаем наблюдение."]

*Смотрим на этой неделе:*
— [конкретный индикатор]
— [2–3 максимум]
```

### Sections that may be skipped on quiet weeks

- *Корзины* — если 0 ATC за неделю: одна строка `Корзин за неделю не было.`
- *Каналы* — если ни один не сдвинулся больше чем на 20%, пропустить.
- *Кандидаты на расширение* — обязательный, но при отсутствии находок — одна строка.

## Monthly template (≤1500 words)

Monthly follows the same no-table rule. Use the same inline list patterns.

```
*Pinkspink — [Month Year]*

*[Headline 2–3 фразы — главное за месяц.]*

*Главные числа:*
Сессии: N (пред. M, ±X%)
Корзины: N (пред. M, ±X%)
Checkout: N (пред. M, ±X%)
Покупки: N
Уникальных посетителей: N

*Динамика по неделям:*
```
W17 (20–26.04): N сес · N ATC
W18 (27.04–3.05): N сес · N ATC
W19 (4–10.05): N сес · N ATC
W20 (11–17.05): N сес · N ATC
```

*Каналы:*
Social: N сес (пред. M, ±X%)
Paid: N сес (пред. M, ±X%)
Direct: N сес (пред. M, ≈флэт)
Organic: N сес (пред. M, ±X%)

*География:* 🇯🇵 Япония N · 🇺🇸 США N · ... [топ-5, остальные N стран]

*Воронка и узкие места:*
[2–3 предложения. Главное горлышко. На месячном горизонте уже можно делать выводы.]

*Карточка товара:*
[1–2 предложения. Топ по просмотрам, средняя глубина, время на товаре.]

*Когорты:*
[1–2 предложения. Retention — только на месячном горизонте имеет смысл.]

*Standing hypotheses — статус:*
— [Гипотеза 1]: [подтвердилась / опровергнулась / без изменений]
— ...

*Кандидаты на расширение — итог месяца:*
[Свести недельные находки. Что появлялось 2+ раза = устойчивый сигнал. Что разовое — отбросить.]

*Confounders за месяц:*
[Список с датами если были: VPN-всплески, кампании, деплои, баги.]

*Рекомендации на следующий месяц:*
1. *[Стратегическое действие]* — [мотивация, метрика, срок теста]
2. ...
3. ...

*Что бы посмотрел в следующий раз:*
— [вопрос или разрез]
— [3–5 пунктов]
```

## Если данных нет

Не выдумывай числа. Скажи прямо:

`Данные за [дата] не доступны на момент запроса. Возможные причины: задержка GA4→BigQuery export (обычно до 24ч), проблема с MCP, отсутствие активности.`

И предложи что сделать дальше.

## Делая отчёт впервые

Первый отчёт — устанавливает baseline, не выдаёт рекомендации:
— Опиши текущее состояние без сравнений
— Не выдумывай "просадки" из ничего
— Попроси у пользователя 2–3 решения: пороги аномалий, приоритетные метрики
