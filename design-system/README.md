# Pinkspink Design System

Живая дизайн-система для аналитических дашбордов. Растёт по мере того, как мы доходим до каждой темы в рамках Pinkspink-проекта. Папка переносимая — её можно скопировать в любой следующий проект и использовать как стартовую точку.

## Принципы

- **Шрифт:** Ubuntu Mono (моноширинный, подходит для цифр и таблиц)
- **Токены через CSS-переменные** — никаких bundler'ов, никаких внешних зависимостей; всё работает в любом standalone HTML
- **Источник правды — JSON-файл** в каждом разделе (`tokens.json`); HTML и Markdown файлы рядом — это его визуальная и человекочитаемая проекции
- **Минимум значений, максимум ролей** — один токен может покрывать несколько UI-ролей; имя токена выбирается по самой важной роли
- **Один уровень визуального обрамления.** На одной точке экрана не может быть двух вложенных карточек (`background` + `border-radius` + `box-shadow`). Если компонент-обёртка живёт внутри `.block`, его внутренний wrapper («naked») имеет только функциональные свойства: `overflow`, `max-height`, `position: sticky`, scrollbar styling. Карточка приходит от `.block`. Это правило распространяется на все будущие компоненты дизайн-системы.

## Миграция

Конкретный план внедрения трёх готовых разделов в действующий Pinkspink-дашборд — [`MIGRATION.md`](MIGRATION.md). Четыре фазы (подготовка → typography → charts → controls → верификация), оценка ~6-7 часов в один заход. План не выполнен — открываем когда сядем за внедрение.

## Как использовать в новом проекте

1. **Скопируй папку** `design-system/` в новый проект (или конкретный нужный раздел)
2. **Открой раздел** — внутри найдёшь `tokens.json` (источник правды), `tokens.md` (правила), `*.html` (визуальный референс)
3. **Импортируй токены** в свой код:
   - Если HTML — скопируй блок `:root { --... }` из спецификации в свой `<style>`
   - Если генератор (Python/Node/etc.) — прочитай `tokens.json` и собери `:root` блок программно
4. **Применяй** через `var(--fs-h2)`, `var(--fw-bold)` и так далее. Всегда через переменные, не через хардкод.

## Версионирование

Каждый раздел имеет свою версию (например, `typography v1.0`). Изменения значений токенов = новая мажорная версия. Добавление новых токенов без изменения старых = минорная.
