Tables v0 · обзор

Pinkspink Design System · Таблицы · 2026-04-29

Сейчас короткие таблицы (3-6 строк) живут внутри .h8x12 ячейки 720px высотой → много пустоты. Длинные таблицы потенциально могут «сползать» вниз страницы. Этот файл — обзор + предложение: таблица сама определяет высоту по контенту, при превышении 15 строк — внутренний scroll со sticky-header.

2. Найденные нестыковки

Issue 1
Короткие таблицы не «прижимаются» к контенту
Таблица из 3-6 строк ≈ 150-200px. Контейнер .h8x12 > .cell = 640px. Под таблицей ~440px пустоты. Из-за grid system grid-auto-rows: 80px ячейка форсит высоту. .tbl-compact { align-self: flex-start } только частично решает (выравнивает таблицу к верху, но cell всё равно занимает 640px).
Issue 2
Нет max-height для длинных таблиц
Сейчас все таблицы короткие, но при появлении длинной (например, transactions, top-50 products) не определено: будет ли scroll, какой максимум, есть ли sticky-header. .cell { overflow: auto } прокручивает контейнер целиком, но это побочный эффект, а не правило.
Issue 3
Нет sticky-header при scroll
Когда длинная таблица прокручивается, заголовки колонок уезжают вверх. Пользователь скроллит и не помнит что в каждой колонке. Стандартное решение — thead th { position: sticky; top: 0 }.
Issue 4
.tbl-compact opt-in
Сейчас 7 из 14 таблиц используют этот modifier (компактные строки). Остальные 7 не используют — выглядят разреженно. Должен быть дефолт — все таблицы по дефолту компактные. Если где-то нужно «просторно» (вряд ли) — отдельный modifier.
Issue 5
Высота строки не привязана к шкале контролов
Сейчас td padding: 6px 12px + line-height 1.3 даёт ~25px row. Контролы — 28px (--h-control). Незначительная, но видимая разница. Унификация: row-height 28px = button-height — визуальный ритм через всю UI.

3. Предложение конвенций · Tables v1.0

Цель: таблица сама определяет высоту контейнера, до 15 строк включительно. Свыше — внутренний scroll со sticky-header.

ПараметрСейчасСталоЭффект
Высота строки ~25px (padding 6+13.7+6) 28px (= --h-control) Tables и controls на одном ритме. Через td/th { height: 28px; padding: 0 12px }.
tbl-compact opt-in modifier дефолт Все таблицы компактные. Класс .tbl-compact упраздняется.
Контейнер таблицы живёт внутри .h8x12 .cell (640px фикс) auto-fit (по контенту до 15 строк) Таблица заворачивается в свой .data-table-wrap, без grid-cell-обёртки.
Max-height нет 16 × 28 = 448px 1 header + 15 строк. Дальше — scroll.
Sticky header нет да thead th { position: sticky; top: 0 } — заголовки видны при scroll.
Scrollbar дефолтный браузерный тонкий 6px Не съедает место, не доминирует.

CSS-правила:

:root {
    --row-h-table: 28px;        /* = --h-control: визуальный ритм */
    --max-rows-table: 15;
}

.data-table-wrap {
    max-height: calc(var(--row-h-table) * (var(--max-rows-table) + 1));  /* 28 × 16 = 448px */
    overflow-y: auto;
    border-radius: var(--r-xl);
    box-shadow: var(--sh-card);
    background: var(--bg-card);
}

.data-table { width: 100%; border-collapse: collapse; }

.data-table th {
    position: sticky; top: 0; z-index: 1;
    background: var(--bg-inverse); color: var(--tx-ondark);
    height: var(--row-h-table);
    padding: 0 var(--sp-3);
    font-size: var(--fs-caption);
    font-weight: var(--fw-semibold);
    text-align: left;
}

.data-table td {
    height: var(--row-h-table);
    padding: 0 var(--sp-3);
    font-size: var(--fs-caption);
    line-height: 1.25;
    border-bottom: 1px solid var(--bg-divider);
    vertical-align: middle;
}

.data-table tr:last-child td { border-bottom: none; }
.data-table tr:hover { background: var(--bg-hover); }
.data-table .highlight { font-weight: var(--fw-bold); color: var(--c-highlight); }

.data-table-wrap::-webkit-scrollbar { width: 6px; }
.data-table-wrap::-webkit-scrollbar-track { background: transparent; }
.data-table-wrap::-webkit-scrollbar-thumb { background: var(--tx-muted); border-radius: var(--r-control); }

6. Вопросы для решения

Q1
Принимаем row-height = 28px?
Совпадает с --h-control. Альтернатива — 24px (плотнее) или 32px (просторнее). 28px — золотая середина и единая шкала с кнопками.
Q2
Max 15 строк до scroll — ок?
15 × 28 = 420px + header 28 = 448px. Альтернативы: 10 (320px), 20 (560px). 15 — типичный размер для «обзорной» таблицы (top-N).
Q3
Sticky-header принимаем?
Когда таблица скроллится, заголовки колонок остаются вверху. Стандартный паттерн (Linear, Notion, Looker, etc.). Принимаем без альтернатив?
Q4
Убрать .tbl-compact класс?
Сейчас 7 из 14 таблиц используют. Если все таблицы становятся компактными по дефолту — modifier не нужен. Удалить класс и убрать его из HTML, или оставить как no-op для backward compat?
Q5
Тонкий scrollbar 6px?
Не съедает место, не доминирует. Альтернатива — дефолтный браузерный (≈12px на macOS). 6px — стандарт для дашбордов.
Q6
Что с .h8x12 в дашборде?
Эта grid-ячейка форсит высоту 720px у блоков с таблицами. С новой системой таблица не нуждается в фикс-высоте. Замена: блок с таблицей использует обычный flow (h2/h3 + .meta + .data-table-wrap), без grid-cell-обёртки. ОК?
