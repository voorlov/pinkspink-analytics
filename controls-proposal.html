2. Найденные нестыковки

Issue 1 · Размеры и веса
Три разных font-size, три разных font-weight на трёх типах кнопок
grain-btn: 13px / regular. tab-btn: 12px / semibold. grain-sq: 12px / bold. Все три — переключатели, но визуально разные. Логики не видно.
Issue 2 · Активное состояние
Два разных паттерна active: с бордером и без
grain-btn: только смена фона + цвета. tab-btn: + ещё border-color меняется. grain-sq: вообще нет active-состояния в коде (визуально не понятно какой grain выбран в шапке).
Issue 3 · Padding вне токенов
tab-btn использует 6px 14px — хардкод
Все остальные кнопки используют var(--sp-*). Один tab-btn выпадает из шкалы 4/8/12/16. Если меняем spacing-токены — tab-btn останется как был.
Issue 4 · Нет focus-состояния
Накл. Tab по клавиатуре — не видно куда упал фокус
Accessibility: при нажатии Tab пользователь должен видеть какой элемент сейчас в фокусе. У текущих кнопок :focus отсутствует — фокус невидимый.
Issue 5 · Нет disabled-состояния
Если понадобится отключить кнопку — нечем визуально
Сейчас в дашборде всё активно, но в будущем будут случаи (например, «дисейблить month-grain если данных меньше 4 недель»). Нужен консистентный паттерн disabled.
Issue 6 · Чекбоксы — браузерные дефолты
Чекбоксы выглядят системно, не вписываются в дизайн
Сейчас <input type="checkbox"> рендерится как стандартный системный квадратик. На моноширинном дашборде смотрится чужеродно. Нужен кастомный стиль — обычно делают через appearance:none + кастомные псевдоэлементы.
Issue 7 · font-family на кнопках
Кнопки наследуют системный sans-serif, не Ubuntu Mono
Это уже решённая проблема — мы зафиксировали в Typography v1.0 правило * { font-family: inherit; }. Контролы получат правильный шрифт когда применим typography.

3. Предложение единой системы

Три типа контролов, единые правила состояний, опора на токены. Никаких хардкод-значений вне Typography v1.0 / spacing.

.btn — Toggle button (заменяет grain-btn И tab-btn)

Один тип переключателя для всех мест: фильтры, вкладки, table sort headers и т.д. Default — без бордера (бордер только в focus). Все размеры и веса — из Typography v1.0.

Базовые параметры:

.btn {
    font-family: var(--ff-mono);              /* наследуется */
    padding: var(--sp-1) var(--sp-3);          /* 4×12 (из spacing-токенов) */
    font-size: var(--fs-body);                 /* 13px */
    font-weight: var(--fw-regular);            /* 400 */
    border-radius: var(--r-md);                /* 4px */
    border: 1px solid transparent;             /* резерв под focus-outline без скачков размера */
    transition: background 120ms, color 120ms;
}

.btn-sq — Square button (заменяет grain-sq)

Фиксированный 32×32 для случаев когда нужна «иконочная» компактность (D/W/M в шапке, кнопки сортировки, иконки фильтров). Все остальные правила — те же что у .btn. Добавил active-состояние (его сейчас нет в дашборде).

Только для top-level навигации дашборда (Сводка / Воронки / Аналитика). Просто текст с подчёркиванием — никакой заливки, никакого тёмного фона. Визуально отделяет навигацию от чёрных filter-кнопок.

.tab-bar { border-bottom: 1px solid var(--bg-border); }

4. Соответствие Typography v1.0

Все размеры и веса — из шкалы типографики:

  • .btn: --fs-body 13 / --fw-regular 400
  • .btn-sq: --fs-body 13 / --fw-semibold 600 (квадратные кнопки традиционно полужирные для компактности)
  • .check label: --fs-caption 11 / --fw-regular 400
  • filter-bar .lab: --fs-caption 11 / --fw-regular 400

Все размеры из spacing v1.0: padding var(--sp-1) var(--sp-3) = 4×12. Никаких хардкод-цифр.

5. Принятые решения

Зафиксированы в этой версии файла. Все примеры выше уже сделаны по этим правилам.

Q1 ✓
Два компонента: .btn для фильтров, .tab для навигации
Filter-кнопки (grain, device, период) — заполненный --bg-muted, active = чёрная заливка. Top-level вкладки (Сводка / Воронки / Аналитика) — отдельный .tab: только текст с подчёркиванием при active. Навигация визуально отделена от фильтров.
Q2 ✓
Active = dark fill
background: --bg-inverse; color: --tx-ondark. Самый контрастный паттерн, читается с любого расстояния.
Q3 ✓
Focus = outline 2px + offset 2px
Цвет — --c-channel-social (бренд-фиолетовый). Outline не сдвигает layout. Доступность с клавиатуры.
Q4 ✓
Disabled = opacity 0.4 + cursor not-allowed
Контрол узнаваем, но явно «не активен».
Q5 ✓
Чекбокс checked = чёрный (--bg-inverse)
Заполнение --bg-inverse (#2D3436) с белой галочкой. Не фиолетовый — не отвлекает от данных, нейтрально, в духе моноширинной «инженерности».
Q6 ✓
Filter-bar — Variant A с filter-group
Каждая группа: подпись --fs-caption UPPERCASE серым + 1px полоска снизу + кнопки. Несколько групп side-by-side. Segmented control (B) — отвергнут.
Q7 ✓
Border-radius всех контролов = 2px
Новый токен --r-control: 2px. Применяется к .btn, .btn-sq, .check. Существующие --r-md (4px), --r-lg (6px), --r-xl (8px) остаются для карточек/таблиц.
Q8 ✓
Высота .btn = .btn-sq = 32px
Раньше .btn был ~28px (по padding), .btn-sq — 32px. Стояли рядом в filter-баре с разной высотой → визуальный шум. Теперь оба height: 32px с явным display: inline-flex; align-items: center. Padding у .btn только горизонтальный (0 var(--sp-3)). Centering через flex.

6. Размер контрола — 24 / 28 / 32 px

Три варианта высоты на одинаковом контенте. Меняется только высота — шрифт остаётся --fs-body 13px. Выбери глазами какой ритм лучше: 24px = плотно (для power-user), 28px = баланс, 32px = просторно.
